# Journal Style Presets

Use these presets as starting points. Scientific correctness overrides style.

## Nature / Science style

- Purpose: high-level mechanism, graphical abstract, or conceptual framework.
- Layout: sparse, highly curated, minimal panels.
- Palette: muted neutrals with one restrained accent.
- Text: very short labels; details move to caption.
- Avoid: dense equations, implementation details, decorative gradients.

## Elsevier engineering journal style

- Purpose: system framework, experiment pipeline, cyber-physical coupling, optimization workflow.
- Layout: clear modules, grouped subsystems, explicit data/control arrows.
- Palette: neutral system components plus semantic accents for proposed method and outputs.
- Text: concise but explanatory module labels are acceptable.
- Good for: Applied Energy, Energy, Transportation Research, Reliability Engineering, engineering systems journals.

## IEEE style

- Purpose: algorithm architecture, signal/data flow, model block diagram.
- Layout: compact, grid-aligned, black-and-white robust.
- Palette: very limited color; line style and labels must carry meaning.
- Text: compact technical notation; tensor dimensions can be shown when necessary.
- Avoid: soft shadows, large empty decorative areas, low-contrast pastel fills.

## Graphical abstract style

- Purpose: communicate problem → proposed solution → main outcome in one glance.
- Layout: 3-zone narrative; fewer modules than a method figure.
- Palette: slightly more visual contrast than manuscript figures, but still restrained.
- Text: reader should understand the story within 10 seconds.
- Avoid: full method details, all constraints, all baselines, result tables.

## Default recommendation

If no venue is specified, use the Elsevier engineering journal style for system/optimization papers and IEEE style for model architecture papers.
